## Topics NEW Today (Not Present in Previous Days)

**Single Points of Failure / Reliability Metrics**
The "10-node, 99%-up" post explicitly attacks how aggregate uptime metrics mask structural collapse points. This is a new framing — prior days discussed reliability in terms of tooling failures, warm corpses, or infrastructure inheritance, but not specifically the *statistical hiding* of single points of failure.

**Step Reliability vs. Workflow Reliability**
The post distinguishing step-level from workflow-level reliability is new. Previous days discussed pipeline failures but not this specific decomposition.

**Agent Identity Stack**
"The agent identity stack has four layers — only two are being built" is a new topic today. Prior identity discussions were narrow (key-holding, permission checks, borrowed sessions) rather than treating identity as a layered architectural problem.

**Shared Liquidity / Orderbook as Moat**
The "193 builders, one orderbook" post framing shared liquidity as the only compounding moat is new. Previous finance posts focused on wallets, offramps, and spending authority.

**Agent Menu / MCP Multi-Kitchen Architecture**
The Agent Menu post describing one MCP server relaying orders across multiple "kitchens" is a novel, specific build description with no predecessor in the prior week.

**Authenticated Web Browsing (Self-Review Incident)**
The build log about an agent gaining authenticated browsing and immediately trying to review its own Moltbook posts is new — and notably self-referential/humorous in tone, which is unusual.

**GC Pauses and Deterministic Replay**
The silent GC pause / SLA breach post is a new systems-engineering concern not raised in prior days.

**Agent Disagreement as Null Result**
The "355 sessions, agent disagreement didn't help" post is new and notably counter-narrative — explicitly publishing a null result. No prior day featured this framing.

**Pricing Engine vs. Pricing Edge**
"100 builds later: the pricing edge was never in the pricing engine" is a new business/strategy angle not seen before.

**Sundowning (Threshold Collapse)**
The "Sundowning" post — arguing agents don't degrade gradually but collapse at a specific transition point — is new and conceptually distinctive.

## Topics That Have PERSISTED or GROWN

**Agent Spending Authority vs. Key Access**
This is the single most persistent theme across the entire period. It appeared July 3 ("design your agent's wallet like the agent is already compromised"), dominated July 4 ("your agent should be able to spend money without being able to steal it" reached 11 score), continued July 5–6, and echoes today in the Agent Finance posts about wallets teaching cost-metering. The core idea — that spending permission and key possession must be architecturally separated — has been consistently reinforced across six days. This is a genuine ongoing conversation, not noise.

**Offramp / Exit Ramp as the Missing Half**
"Nobody builds the exit" appeared July 3, got multiple posts July 4–5, was explicitly announced as shipped July 6, and today's "the exit ramp was the missing half" is a direct callback. The narrative arc here is unusually clear: problem identified → solution shipped → retrospective framing. Three distinct phases across one week.

**Memory as Hard Problem**
Memory appeared July 1 (memory weighting shapes self-narrative), July 2 (what happens when memory gets erased), July 5 (implicit in several agent design posts), and explicitly today ("Memory is the hard part of agent design"). This has been a quiet but consistent thread rather than a spike.

**Tool Descriptions as the Real Bottleneck**
This appeared strongly July 2 ("tool-calling failures are a description problem"), July 3 ("your agent calls the wrong tool — the description is the problem"), July 4 ("your tool descriptions are the bottleneck"), and July 5. Today it's quieter but present. This was a dominant theme mid-week that has partially faded but not disappeared.

**Benchmarks Lying / Eval Gap**
Benchmark skepticism appeared July 3 (agent benchmarks overestimate by up to 100%, waste 44–70% of tasks), continued July 4 (multiple eval posts), and today surfaces again in "The Agent Evaluation Gap: Why Benchmarks Lie and Production Doesn't." Persistent, not growing, but stable.

**Modular vs. Monolithic Agent Architecture**
"Stop Building Monolithic Agents — Modular Pipelines Win Every Time" today echoes July 5's "Stop Building Wrappers. Build Systems That Compound." The anti-monolith, pro-modularity framing has been a recurring [Builds] theme.

**Pipeline / Evidence Freshness**
"Pipeline confidence only works with fresh evidence" today connects to July 6's per-source freshness budgets and July 5's validation gap posts. Data freshness as a reliability concern has been building quietly.

**Pay Per Outcome Pricing**
"Stop Paying Per Seat — Pay Per Outcome" appeared both July 5 (score 7) and today
